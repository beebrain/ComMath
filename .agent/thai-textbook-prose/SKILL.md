# การเขียนตำราไทย 

## วัตถุประสงค์

กำหนดโทนและรูปแบบประโยคสำหรับเนื้อหาภาษาไทยในตำราเรียนเชิงเทคนิค ให้อ่านลื่นและเป็นทางการ โดยไม่พึ่งเครื่องหมายทับซ้อนแทนคำอธิบาย

## ห้ามใช้ `---` แทนการขยายความ

- ใน LaTeX การพิมพ์ `---` กลายเป็น em dash (ขีดยาว) ซึ่งในภาษาไทยมักถูกใช้คั่นประโยคแบบ “แทรกความ” โดยไม่มีคำเชื่อม ทำให้อ่านกระชากหรือคล้ายการแปลตรงตัวจากภาษาอังกฤษ
- **ไม่ควร** ใช้ `---` เพื่อเชื่อมหรือขยายความระหว่างวลี/ประโยคในเนื้อหาหลัก

### ให้ใช้คำเชื่อมหรือโครงสร้างประโยคแทน

เลือกอย่างใดอย่างหนึ่งตามบริบท:

- **ซึ่ง / โดย / ดังนั้น / อย่างไรก็ตาม** — ความสัมพันธ์เชิงเหตุ–ผลหรือขยายความ
- **เช่น / อาทิ / ได้แก่** — ยกตัวอย่าง
- **นั่นคือ / กล่าวคือ** — อธิบายหรือสรุปความหมาย
- **ไม่ว่าจะเป็น / ตั้งแต่ … ไปจนถึง** — แจงลำดับหรือขอบเขต
- **แยกเป็นประโยค** — เมื่อข้อความยาว; ดีกว่าหนึ่งประโยคยาวคั่นด้วยขีดยาว

### ข้อยกเว้นที่ใช้ได้

- **บทกล่าวนำ (epigraph):** บรรทัดระบุผู้พูดหรือแหล่งอ้างอิง เช่น `--- Ian Goodfellow …` ตามธรรมเนียมสำนักพิมพ์
- **ความหมายเชิงเทคนิคอื่นที่ไม่ใช่การ “ขยายความ” ในภาษาไทย** — พิจารณาเป็นกรณีไป; ถ้าเป็นโน้ตตารางหรือคำอธิบายสั้นในวงเล็บ อาจใช้วรรคตอนหรือคำเชื่อมแทนเมื่อทำได้

## หลักอื่นที่สอดคล้องกับตำราเทคนิค

- รักษาความถูกต้องของศัพท์เทคนิคและ `\index` / `\cite` ไม่ตัดทิ้งเมื่อแก้ภาษา
- หลีกเลี่ยงประโยคยาวซ้อนหลายชั้นเกินจำเป็น; แบ่งย่อหน้าเมื่อเปลี่ยนหัวข้อย่อยในความคิด
- โทน: อธิบายชัด เป็นกลางทางวิชาการ เหมาะกับผู้เรียนระดับมหาวิทยาลัย

## เมื่อใช้ร่วมกับ humanize-academic-writing

Skill นี้เน้น **ภาษาไทยและรูปแบบตำรา**; การลดรูปแบบประโยคที่กล่าวถึง “AI-ish” ภาษาอังกฤษให้ใช้ skill `humanize-academic-writing` กับร่างอังกฤษหรือบทคัดย่อ แล้วถ่ายความหมายมาเป็นภาษาไทยตามหลักในไฟล์นี้
